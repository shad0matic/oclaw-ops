## 10. YannDecoopman & im_bulent on Agent Memory

- **Tweets (Combined):** Two tweets discussing effective, simple methods for giving OpenClaw agents long-term memory. The core idea is using a markdown file (`SOUL.md` or `memory.md`) that the agent reads before every task and updates after. One tweet suggests indexing this with Voyage AI for semantic search.
- **Business/Money Opportunities:**
    - **Agent Memory Solutions:** Offer pre-built, robust memory systems for OpenClaw users, potentially packaging the Obsidian + Voyage AI setup as an easy-to-deploy service.
- **Actionable Insights:**
    - Simple solutions are often the most effective. A plain text file can be a surprisingly robust long-term memory solution for an AI agent.
    - Semantic search is key for making a large memory store useful.
- **Tools or Resources Mentioned:**
    - **Obsidian:** A note-taking app that uses local markdown files.
    - **Voyage AI:** A semantic indexing and search API.
- **Potential Projects:**
    - **Create a "Memory Skill" for OpenClaw:** An easy-to-install skill that automatically handles reading/writing to a memory file and querying a Voyage AI index.
